# Uitleg K6

K6 is een Javascript performancetest tool voor zowel lokaal als cloud. Doordat de performancetesten in Javascript (ES6) geschreven worden maakt dit het voor programmeurs en testers toegankelijk. Wat zijn de standaard elementen in een performancetest script?

Er zijn een viertal standaard elementen/functies gedefinieerd voor het bouwen van een performancetest met K6. 

- <b>Init:</b> Hier komen alle imports van libraries die gebruikt worden. Dit kunnen custom libraries zijn of libraries van K6 zelf.
- <b>Setup:</b> De setup function wordt gebruikt om voorbereidende taken uit te voeren.
- <b>Default:</b> De functie "export default function" is de functie dat het http request afhandeld.
- <b>Teardown:</b> Deze functie is bedoeld om de test netjes af te sluiten.

```javascript
//init
import http from 'k6/http'

//setup
export function setup() {

}

//VU code
export default function() {
    http.get("http://www.foobar.com")
}

//teardown
export function teardown() {

}
```

Parameters kunnen zowel via terminal als in het script worden meegegeven. Voorbeelden hiervan zijn: Virtual users (vus) en duration. Beide parameters worden anders gedefinieerd. Vus is vast getal zoals (1, 3, 8 of 12) duration wordt uitgedrukt in tijd dus het aantal seconden, minuten of uren. Voorbeelden hiervan zijn (10s, 8m, 1h).

Terminal command voor uitvoeren test begint altijd met k6 - run - parameters - foo.js

<b>Voorbeeld:</b>

```bash
k6 run --vus 10 --duration 30s foo.js
```
